# /next-stage

분석·설계 파이프라인의 단계 진행을 관리한다.

수행할 일:

1. 현재 run 디렉터리(`demo/<run>/`)에 어떤 산출물이 이미 생성되었는지 확인한다.
   파일 순서: 01-system-request → 02-requirements → 03-use-cases → 04-structure-dfd → 05-erd → 06-adr
2. 가장 최근 산출물이 **검증을 통과**했고 **사용자 승인을 받았는지** 확인한다.
   - 아직 승인 전이면: 산출물 요약을 보여주고 승인을 요청한다 (다음으로 넘어가지 않는다).
   - 승인되었으면: 다음 단계의 인터뷰를 시작한다 (`spec-pipeline` 스킬의 단계 루프를 따른다).
3. 모든 단계(S6)가 끝났으면, 완료를 알리고 `demo/<run>/`의 산출물 목록을 정리해 보여준다.

규칙: 승인 게이트(R2)를 건너뛰지 않는다. 검증(R3) 미통과 산출물 위에서 다음 단계로 진행하지 않는다.
