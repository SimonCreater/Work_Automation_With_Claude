# 요구사항 정의 (Requirements Definition)

> 모든 요구사항은 **검증 가능한 서술**로 작성한다. ("빠르다" X → "검색 응답 3초 이내" O)
> 각 항목은 고유 ID를 가진다. 기능=FR-n, 비기능=NFR-n.

## 기능 요구사항 (Functional Requirements)

| ID | 요구사항 (검증 가능한 서술) | 우선순위 |
|----|------------------------------|----------|
| FR-1 | 시스템은 표준 계정과목 형식의 3개년 재무제표(재무상태표·손익계산서)를 업로드받아 매출·영업이익·부채비율·이자보상배율 등 핵심 재무지표를 자동 산출해야 한다. | 상 |
| FR-2 | 시스템은 입력된 산업 동향 자료와 재무지표를 근거로 신용등급 "의견(초안)"과 최소 3개 이상의 리스크 요인을 생성하고, 각 항목마다 출처(재무제표 라인/산업자료 위치)를 명시해야 한다. | 상 |
| FR-3 | 시스템은 생성된 신용메모 초안에 대해 심사역이 단계별(초안 검토 → 코멘트 → 확정 요청)로 승인/반려할 수 있는 워크플로를 제공해야 한다. | 상 |
| FR-4 | 시스템은 심사역이 반려 시 사유를 입력하면 해당 섹션만 재생성하고, 변경 이력을 버전으로 보존해야 한다. | 중 |

## 비기능 요구사항 (Non-Functional Requirements)

> 최소 한 가지 카테고리 이상을 다룬다: 성능 / 보안 / 사용성 / 신뢰성 / 규제·준수.

| ID | 카테고리 | 요구사항 (검증 가능한 서술) |
|----|----------|------------------------------|
| NFR-1 | 성능 | 3개년 재무제표 입력 후 신용메모 초안 생성까지 90초 이내에 완료해야 한다. |
| NFR-2 | 보안 | 기업 재무정보는 망분리된 내부 환경에서만 처리하며, 외부 네트워크로 어떤 입력 데이터도 전송하지 않아야 한다. |
| NFR-3 | 규제·준수 | 모든 등급 의견·리스크 판단은 근거 출처와 함께 감사 로그로 보존되어, 사후에 판단 경로를 100% 추적할 수 있어야 한다. |
