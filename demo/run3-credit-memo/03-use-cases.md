# 유스케이스 (Use Cases)

> 이론(ch04) fully-dressed 형식. **2건 이상**, 각 유스케이스는 정상 흐름과 함께
> **대안/예외 흐름을 반드시 포함**한다. 시퀀스상 의미가 통하도록 작성한다.

---

## UC-1: 신용메모 초안 생성하기

- **ID**: UC-1
- **Priority**: High
- **Primary Actor**: 여신 심사역
- **Trigger**: 심사역이 신청 기업의 3개년 재무제표와 산업 동향 자료를 업로드하고 "초안 생성"을 요청한다.
- **Trigger Type**: External
- **Description**: 심사역이 입력한 재무·산업 자료를 바탕으로 시스템이 신용등급 의견·리스크 요인·근거 출처가 포함된 신용메모 초안을 생성한다.

### Preconditions (사전 조건)
- 심사역이 인증된 상태로 시스템에 접근해 있다.
- 신청 기업 식별정보가 등록되어 있다.

### Normal Flow (정상 흐름)
1. 심사역이 3개년 재무제표 파일과 산업 동향 자료를 업로드한다.
2. 시스템이 재무제표를 파싱해 핵심 재무지표(부채비율·이자보상배율 등)를 산출한다.
3. 시스템이 재무지표와 산업 동향을 결합해 신용등급 의견 초안과 리스크 요인을 생성한다.
4. 시스템이 각 판단에 근거 출처를 부착해 신용메모 초안을 화면에 제시한다.
5. 초안과 산출 지표가 심사 건에 버전 1로 저장된다.

### Alternate / Exception Flow (대안·예외 흐름)
- 1a. 업로드된 재무제표가 표준 계정과목 형식이 아니거나 연도가 3개년 미만이면, 시스템은 누락·불일치 항목을 표시하고 초안 생성을 중단한 뒤 재업로드를 요청한다.
- 2a. 특정 계정과목 값이 비어 있어 지표 산출이 불가능하면, 시스템은 해당 지표를 "산출 불가"로 표기하고 영향을 받는 의견 영역을 보수적으로 처리한다.
- 3a. 산업 동향 자료가 첨부되지 않으면, 시스템은 산업 리스크 섹션을 "자료 미제출"로 명시하고 재무 기반 의견만 생성한다.

### Postconditions (사후 조건)
- 근거 출처가 부착된 신용메모 초안과 산출 지표가 버전으로 저장된다.

## UC-2: 초안을 단계별로 검토·승인하기

- **ID**: UC-2
- **Priority**: High
- **Primary Actor**: 여신 심사역
- **Trigger**: 심사역이 생성된 신용메모 초안을 열어 검토를 시작한다.
- **Trigger Type**: External
- **Description**: 심사역이 초안을 섹션 단위로 검토하며 승인 또는 사유와 함께 반려하고, 반려된 섹션은 재생성되어 최종 확정 요청까지 진행된다.

### Preconditions (사전 조건)
- UC-1을 통해 신용메모 초안(버전 1 이상)이 존재한다.

### Normal Flow (정상 흐름)
1. 심사역이 초안의 등급 의견·리스크 요인·근거 섹션을 순서대로 검토한다.
2. 심사역이 각 섹션을 승인한다.
3. 모든 섹션이 승인되면 시스템이 "확정 요청" 상태로 전환한다.
4. 시스템이 승인 이력과 최종 버전을 감사 로그에 기록한다.

### Alternate / Exception Flow (대안·예외 흐름)
- 2a. 심사역이 특정 섹션을 반려하고 사유를 입력하면, 시스템은 해당 섹션만 재생성해 새 버전으로 제시하고 검토 단계로 되돌린다.
- 3a. 근거 출처가 비어 있는 항목이 남아 있으면, 시스템은 확정 요청 전환을 막고 해당 항목을 강조 표시한다.

### Postconditions (사후 조건)
- 승인된 최종 버전과 단계별 승인·반려 이력이 감사 로그에 보존된다.
