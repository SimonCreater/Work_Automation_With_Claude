# 요구사항 정의 (Requirements Definition)

## 기능 요구사항 (Functional Requirements)

| ID | 요구사항 (검증 가능한 서술) | 우선순위 |
|----|------------------------------|----------|
| FR-1 | 시스템은 트레이너가 자신의 예약 가능 시간대를 요일·시각 단위로 등록·수정할 수 있어야 한다 | 상 |
| FR-2 | 시스템은 회원이 트레이너와 가능 시간대를 선택해 PT 예약을 생성할 수 있어야 한다 | 상 |
| FR-3 | 시스템은 동일 트레이너·동일 시간대에 예약이 이미 있으면 새 예약 생성을 거부해야 한다 | 상 |
| FR-4 | 시스템은 회원이 본인 예약을 예약 시작 N시간 전까지 취소할 수 있어야 한다 | 중 |
| FR-5 | 시스템은 예약 시작 전 정해진 시점에 회원에게 리마인더를 1회 발송해야 한다 | 중 |
| FR-6 | 시스템은 예약 시간 종료 후 출석/노쇼 여부를 기록할 수 있어야 한다 | 중 |

## 비기능 요구사항 (Non-Functional Requirements)

| ID | 카테고리 | 요구사항 (검증 가능한 서술) |
|----|----------|------------------------------|
| NFR-1 | 사용성 | 회원은 스마트폰 웹에서 3번 이내의 탭으로 예약을 완료할 수 있어야 한다 |
| NFR-2 | 신뢰성 | 동시에 같은 시간대 예약 시도가 들어와도 확정 예약은 1건만 생성되어야 한다 |
| NFR-3 | 보안 | 회원·트레이너 연락처는 본인과 점장만 조회할 수 있어야 한다 |
