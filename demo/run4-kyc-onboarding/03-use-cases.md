# 유스케이스 (Use Cases)

---

## UC-1: 신규 고객 온보딩 서류 자동 심사하기

- **ID**: UC-1
- **Priority**: High
- **Primary Actor**: 온보딩 운영 담당자
- **Trigger**: 신규 고객이 온보딩 서류 일체를 제출(업로드)한다.
- **Trigger Type**: External
- **Description**: 제출된 서류에서 핵심 항목을 추출하고 제재·PEP·AML 리스트와 대조하여 위험등급이 부여된 심사 케이스를 생성한다.

### Preconditions (사전 조건)
- 고객이 온보딩 절차를 시작하고 필수 서류 유형이 정의되어 있다.
- 제재/PEP/AML 워치리스트의 최신 버전이 시스템에 적재되어 있다.

### Normal Flow (정상 흐름)
1. 담당자가 고객이 제출한 온보딩 서류를 시스템에 업로드한다.
2. 시스템이 각 서류에서 신원·사업자·소득 핵심 항목을 추출한다.
3. 시스템이 추출된 신원 정보를 제재·PEP·AML 워치리스트와 대조한다.
4. 시스템이 대조 결과와 추출 항목을 근거로 위험등급(저/중/고)을 산정한다.
5. 시스템이 모든 근거·출처·시점을 포함한 심사 케이스를 리뷰 큐에 생성한다.
6. 담당자에게 케이스 생성 완료가 통지된다.

### Alternate / Exception Flow (대안·예외 흐름)
- 2a. 서류 항목 추출에 실패하거나 필수 항목이 누락된 경우, 시스템은 해당 건을 '추가 서류 요청' 상태로 분류하고 누락 항목 목록과 함께 담당자에게 통지한다.
- 3a. 워치리스트가 갱신 실패로 만료(stale)된 경우, 시스템은 대조를 중단하고 해당 건을 '대조 보류'로 표시하며 운영자에게 경고를 발생시킨다.
- 4a. 강한 매칭 후보(고유사도)가 발견된 경우, 시스템은 위험등급을 '고'로 강제 상향하고 케이스에 경보 플래그를 표시한다.

### Postconditions (사후 조건)
- 온보딩 건은 '심사 대기' 또는 '추가 서류 요청' 상태의 케이스로 존재한다.
- 추출·대조·등급 산정의 모든 근거가 감사 로그에 기록된다.

## UC-2: 심사 케이스 검토 및 결정하기

- **ID**: UC-2
- **Priority**: High
- **Primary Actor**: 준법감시 심사역
- **Trigger**: 리뷰 큐에 검토 대기 케이스가 존재한다.
- **Trigger Type**: External
- **Description**: 심사역이 자동 생성된 케이스의 매칭 후보와 근거를 검토하여 승인·반려·보류를 결정하고 사유를 기록한다.

### Preconditions (사전 조건)
- UC-1을 통해 심사 케이스가 생성되어 리뷰 큐에 존재한다.
- 심사역에게 해당 케이스를 검토할 권한(RBAC)이 부여되어 있다.

### Normal Flow (정상 흐름)
1. 심사역이 리뷰 큐에서 케이스를 선택한다.
2. 시스템이 추출 항목·매칭 후보·위험등급 근거·원본 서류를 한 화면에 표시한다.
3. 심사역이 매칭 후보의 진위(true/false positive)를 판단한다.
4. 심사역이 승인·반려·보류 중 하나를 결정하고 사유를 입력한다.
5. 시스템이 결정·사유·결정자·시점을 케이스 이력과 감사 로그에 기록한다.
6. 케이스 상태가 결정값에 따라 갱신된다.

### Alternate / Exception Flow (대안·예외 흐름)
- 3a. 심사역이 제시된 근거만으로 판단이 불충분하다고 보면, 케이스를 '추가 서류 요청'으로 전환하고 요청 항목을 지정한다.
- 4a. 위험등급이 '고'인 케이스를 승인하려는 경우, 시스템은 상위 결재자(2차 승인)의 추가 승인을 요구한다.

### Postconditions (사후 조건)
- 케이스는 승인·반려·보류·추가서류요청 중 하나의 최종 상태를 가진다.
- 결정 사유와 결정자가 감사 로그에 불변으로 기록된다.
