# 요구사항 정의 (Requirements Definition)

## 기능 요구사항 (Functional Requirements)

| ID | 요구사항 (검증 가능한 서술) | 우선순위 |
|----|------------------------------|----------|
| FR-1 | 시스템은 사용자가 입력한 종목명 또는 6자리 종목코드로 대상 기업을 식별하고, 동명 후보가 둘 이상이면 목록을 제시해 선택받아야 한다 | 상 |
| FR-2 | 시스템은 선택된 기업의 최근 1년 공시를 공식 전자공시에서 수집하고, 공시 유형·일자·제목을 목록화해야 한다 | 상 |
| FR-3 | 시스템은 최근 4개 분기 실적(매출·영업이익·순이익)을 표 형태로 정리해야 한다 | 상 |
| FR-4 | 시스템은 최근 30일 주요 뉴스를 수집해 일자·매체·제목과 3줄 이내 요약을 제공해야 한다 | 중 |
| FR-5 | 시스템은 수집 자료를 바탕으로 투자 포인트 3건 이상과 리스크 3건 이상을 포함한 리서치 메모 초안을 생성해야 한다 | 상 |
| FR-6 | 시스템은 생성된 메모에 출처 링크와 "초안·투자 자문 아님" 면책 문구를 자동 첨부해야 한다 | 중 |

## 비기능 요구사항 (Non-Functional Requirements)

| ID | 카테고리 | 요구사항 (검증 가능한 서술) |
|----|----------|------------------------------|
| NFR-1 | 성능 | 종목 입력부터 메모 초안 생성까지 95퍼센타일 기준 3분 이내에 완료해야 한다 |
| NFR-2 | 신뢰성 | 메모의 모든 수치·인용은 원문 출처 링크를 1:1로 가져야 하며, 출처 없는 수치는 표기하지 않는다 |
| NFR-3 | 규제·준수 | 시스템은 매매를 실행하지 않으며 모든 산출물에 투자 자문이 아님을 명시해야 한다 |
| NFR-4 | 보안 | 외부 데이터 출처 인증키는 코드·산출물에 평문으로 저장하지 않는다 |
