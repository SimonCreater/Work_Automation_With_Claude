# 협의 세션 로그 (축어) — run1 (종목 리서치 어시스턴트)

> **본인 아이디어 / 사람이 직접 답한 실제 협의 세션의 축어 기록이다.**
> 에이전트 질문 → 사람 답변 → 산출물 Write → **검증 훅 stderr 원문** → 승인 한 줄을
> 일어난 그대로 옮겼다. 이 로그 하나로 R1(에이전트 선질문)·R2(단계별 승인 게이트)·
> R3(기계적 검증, 실패→자가수정 1회 포함)이 실제로 동작했음을 확인할 수 있다.
> (검증 재현은 `bash scripts/selftest.sh` 또는 GitHub Actions 로그 참조)

## 시작 — 아이디어 한 문단 (사람 입력)

> **사용자:** "종목명만 넣으면 공시·실적·뉴스를 요약해 리서치 메모 초안을 만들어주는 도구를 만들고 싶어"

> **에이전트:** 좋습니다. 빈 양식을 드리지 않고 제가 먼저 몇 가지 여쭤보며 정리하겠습니다.
> 한 번에 한 단계씩, 2~4개 질문만 드릴게요. 먼저 '시스템 요청서'부터 시작합니다. (R1)

## S1. 시스템 요청서

> **에이전트 질문:**
> 1. 이 도구를 가장 필요로 하는 사람은 누구인가요? (후원자)
> 2. 지금은 이 일을 어떻게 하고 있고, 무엇이 가장 불편한가요?
> 3. 잘 되면 무엇이 좋아지나요? 가능하면 수치로요.
> 4. 반드시 지켜야 할 제약(법규·데이터 출처 등)이 있나요?

> **사용자:** 증권사 리서치센터 애널리스트요. 한 종목 초기 리서치 자료 모으는 데 종목당 3시간쯤
> 걸려요. 공시·실적·뉴스를 여기저기서 손으로 긁어모으느라. 이걸 30분 안으로 줄이고 싶어요.
> 제약은 — 공식 공시(DART 같은)만 쓰고, 절대 자동매매는 안 돼요. 결과는 '초안'이어야 하고요.

> **에이전트:** 범위를 좁히기 위해 하나만 더요. 1차 버전은 국내 상장 주식만으로 한정할까요?
> 목표주가 자동 산정은 규제 리스크가 있어 보통 빼는데, 제외로 둘까요?

> **사용자:** 네, 국내 KOSPI/KOSDAQ만. 목표주가 자동 산정은 빼주세요. (← ADR 후보로 포착)

→ `01-system-request.md` Write.

```
$ (PostToolUse 훅 자동 실행)
[검증 통과] 01-system-request.md (system-request)
```

> **에이전트 [승인 게이트]:** 후원자=리서치 애널리스트, 니즈=수집시간 3h→30m 단축,
> 제약=공식 공시만·자동매매 금지·국내주식 한정으로 정리했습니다. 이대로 확정하고
> 요구사항 단계로 갈까요? (승인 / 수정)

> **사용자:** 승인. (R2 — 게이트 통과)

> 인증키 평문 저장 금지도 보안 NFR로 추가할까요? (보통 넣습니다)

> **사용자:** 네 넣어주세요.

## S3. 유스케이스  ← **R3 검증 실패가 실제로 발생한 단계**

> **에이전트 질문:** 핵심 시나리오 2개와, 각각 '시작 이벤트'를 알려주세요.
> 그리고 잘못될 수 있는 경우(예: 동명 기업, 출처 장애)도 떠오르는 대로요.

> **사용자:** 1번은 종목 넣으면 메모 초안 만드는 거. 2번은 관심 종목에 중요 공시 뜨면 알림.

**[1차 생성]** 에이전트가 UC-1(메모 생성)·UC-2(공시 모니터링)을 작성했으나, 정상 흐름만 적고
예외 흐름을 빠뜨린 채 Write 했다.

```
$ (PostToolUse 훅 자동 실행)
[검증 실패] 03-use-cases.md (use-cases) — 1건 위반:
  - 각 유스케이스에 대안/예외 흐름이 반드시 필요합니다 (정상 흐름만 있으면 안 됨). (현재 0건)
위 항목을 수정한 뒤 다시 저장하세요. 통과 전에는 다음 단계로 진행하지 않습니다.
```

**[자가 수정]** 에이전트가 사람에게 다시 묻지 않고도, 검증 피드백을 받아 예외 흐름을 보강했다:
- UC-1: 동명 기업 다건(2a) / 공시 출처 장애(4a) / 최근 뉴스 0건(5a)
- UC-2: 당일 신규 공시 없음(2a) / 알림 전송 실패 재시도(5a)
→ 다시 Write.

```
$ (PostToolUse 훅 자동 실행)
[검증 통과] 03-use-cases.md (use-cases)
```

> **에이전트 [승인 게이트]:** UC-1(메모 생성)·UC-2(공시 모니터링), 각각 정상 + 예외 흐름까지
> 포함했습니다. 1차에는 예외 흐름이 없어 검증이 막았고, 보강 후 통과했습니다. 승인하실까요?

> **사용자:** 승인.

> **메모:** 정상 흐름만으로는 승인 게이트에 **도달조차 못 했다.** 사람(승인)보다 기계(검증)가
> 먼저 결함을 걸러낸, R3가 R2 앞단에서 동작한 실증 사례다.
