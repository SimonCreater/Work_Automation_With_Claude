# 유스케이스 (Use Cases)

---

## UC-1: 종목 리서치 메모 초안 생성

- **ID**: UC-1
- **Priority**: High
- **Primary Actor**: 애널리스트
- **Trigger**: 애널리스트가 특정 종목의 초기 리서치를 위해 종목명을 입력한다
- **Trigger Type**: External
- **Description**: 애널리스트가 종목을 입력하면 시스템이 공시·실적·뉴스를 수집·요약해 리서치 메모 초안을 생성한다.

### Preconditions (사전 조건)
- 애널리스트가 시스템에 로그인되어 있다
- 외부 공시·뉴스 데이터 출처가 정상 응답한다

### Normal Flow (정상 흐름)
1. 애널리스트가 종목명 또는 종목코드를 입력한다
2. 시스템이 대상 기업을 식별하고 확인을 요청한다
3. 애널리스트가 대상 기업을 확정한다
4. 시스템이 최근 공시·분기 실적·최신 뉴스를 수집한다
5. 시스템이 수집 자료를 요약하고 투자 포인트와 리스크를 정리한다
6. 시스템이 출처 링크와 면책 문구를 포함한 리서치 메모 초안을 화면에 표시한다
7. 애널리스트가 초안을 검토하고 저장한다

### Alternate / Exception Flow (대안·예외 흐름)
- 2a. 동명 기업이 둘 이상이면 시스템이 후보 목록을 제시하고 애널리스트가 하나를 선택한다
- 4a. 공시 데이터 출처가 응답하지 않으면 시스템이 수집 실패를 알리고, 사용 가능한 자료만으로 진행할지 묻는다
- 5a. 최근 뉴스가 0건이면 시스템이 "해당 기간 주요 뉴스 없음"을 명시하고 공시·실적 기반으로만 요약한다

### Postconditions (사후 조건)
- 리서치 메모 초안이 출처와 면책 문구를 포함한 상태로 저장된다

## UC-2: 보유 종목 공시 모니터링 알림

- **ID**: UC-2
- **Priority**: Medium
- **Primary Actor**: 애널리스트
- **Trigger**: 매일 장 마감 후 정해진 시각이 되면 모니터링 작업이 실행된다
- **Trigger Type**: Temporal
- **Description**: 애널리스트가 등록한 관심 종목에 신규 중요 공시가 올라오면 시스템이 이를 감지해 요약 알림을 보낸다.

### Preconditions (사전 조건)
- 애널리스트가 관심 종목을 1개 이상 등록해 두었다

### Normal Flow (정상 흐름)
1. 정해진 시각에 시스템이 관심 종목 목록을 불러온다
2. 시스템이 각 종목의 당일 신규 공시를 조회한다
3. 시스템이 신규 공시 중 중요 유형(실적·합병·유상증자 등)을 선별한다
4. 시스템이 선별된 공시를 요약해 알림 메시지를 구성한다
5. 시스템이 애널리스트에게 요약 알림을 전송한다

### Alternate / Exception Flow (대안·예외 흐름)
- 2a. 당일 신규 공시가 전혀 없으면 시스템은 알림을 보내지 않고 작업을 종료한다
- 5a. 알림 전송이 실패하면 시스템이 재시도하고, 재시도도 실패하면 다음 실행 시 미발송분을 합쳐 보낸다

### Postconditions (사후 조건)
- 중요 신규 공시가 있으면 애널리스트가 요약 알림을 수신한 상태가 된다
