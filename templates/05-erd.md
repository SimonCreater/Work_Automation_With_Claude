# 데이터 모델 — 개체-관계 다이어그램 (ERD)

> 이론(ch06). 엔티티·속성·관계를 Mermaid erDiagram으로 표현한다.
> 규칙: 각 엔티티는 식별자(PK)를 가진다. 관계는 동사구 이름과 카디널리티(1:1 / 1:N / M:N)를 갖는다.
> DFD의 데이터 스토어와 대체로 대응되어야 한다(balance).

## ERD

```mermaid
erDiagram
    ENTITY_A ||--o{ ENTITY_B : "{{ 관계 동사구 }}"
    ENTITY_A {
        type {{ 속성명_PK }} PK
        type {{ 속성명 }}
    }
    ENTITY_B {
        type {{ 속성명_PK }} PK
        type {{ 외래키_FK }} FK
        type {{ 속성명 }}
    }
```
